# Security policy

## Supported versions

Security fixes are provided for the latest released minor version. Until the
first stable release, fixes are applied to the default branch and included in
the next release.

## Reporting a vulnerability

Please use GitHub's **Report a vulnerability** private reporting feature on the
Security tab of this repository. Include affected versions, reproduction steps,
impact, and any suggested mitigation.

Do not disclose the issue publicly until a fix or coordinated disclosure plan
is available. You should receive an acknowledgement within seven days. If the
report is accepted, maintainers will share progress and coordinate credit.

`oss-ready` only reads local repository paths and does not intentionally make
network requests or execute code from the scanned repository.
